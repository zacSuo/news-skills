# 百度「通用翻译」配置说明（解决 52003 未授权）

本项目的周报翻译使用的是**百度翻译开放平台 - 通用翻译 API**（接口 `/api/trans/vip/translate`），需要的是**通用翻译应用的 APP ID + 密钥**，不是「API Key 管理」里创建的那种 API Key。

## 若出现 52003 UNAUTHORIZED USER

说明当前填写的 APPID/密钥**没有通用翻译接口的权限**。常见原因：

1. **未开通通用翻译**（最常见）：个人中心有 APPID/密钥，但**没有对该应用开通「通用翻译」服务**。必须在控制台找到「产品服务」/「翻译服务」→ **「通用翻译」** → 点击 **「立即开通」** 或 **「开通」**，并选择/绑定到你的应用。
2. **用错了凭证**：复制的是「API Key 管理」里为**大模型翻译**创建的 API Key，那种 Key 只能调大模型接口，不能调通用翻译。
3. **接口域名**：若开通无误仍 52003，可尝试旧版接口：在 .env 增加 `BAIDU_TRANSLATE_HOST=api.fanyi.baidu.com`，再运行 `npm run report:test-translate`。

## 正确获取「通用翻译」APP ID 和密钥

1. 打开 **https://fanyi-api.baidu.com/** 并登录。
2. 进入 **「管理控制台」**（或「开发者中心」）。
3. **开通「通用翻译」**（必须单独开通，不能只靠个人中心的 APPID）：
   - 在「产品服务」或「翻译服务」里找到 **「通用翻译（文本翻译）」**；
   - 点击 **「立即开通」** 或 **「开通」**，选择版本，填写应用名称，提交；
   - 开通成功后，该应用才会具备通用翻译接口权限。
4. **查看应用 ID 和密钥**：
   - 开通成功后，在**该通用翻译应用的详情页**或**申请信息/应用列表**里会看到：
     - **应用 ID**（即 APP ID，一串数字）
     - **密钥**（Secret Key，一长串字符）
   - 有的入口在：控制台 → 我的应用 → 对应「通用翻译」应用 → 应用 ID / 密钥。
5. **填入 .env**：
   - `BAIDU_TRANSLATE_APPID` = 上一步的**应用 ID**
   - `BAIDU_TRANSLATE_KEY` = 上一步的**密钥**（不是 API Key 管理里单独创建的那类 Key）

## 如何区分两种 Key

| 来源 | 用途 | 本项目是否可用 |
|------|------|----------------|
| **通用翻译应用**里的「应用 ID」+「密钥」 | 调用通用翻译接口 vip/translate | ✅ 用这一对 |
| **API Key 管理**里创建的「API Key」 | 大模型文本/文档翻译等 | ❌ 不能用于本脚本 |

配置完成后执行 `npm run report:test-translate` 验证，成功会显示「翻译成功: Hello world -> ...」。
